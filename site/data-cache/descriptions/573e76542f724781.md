## Main Responsibilities

- Conduct penetration testing on web and backend applications, networking components, and crypto services. White, grey box, or black box – you name it!
- Show your reverse engineering skills in finding bugs during code review of cutting-edge security solutions.
- Design and create solutions to handle complex security issues from scratch;
- Develop scripts and security automation tools to enhance penetration testing processes.
- Closely collaborate with web and backend application developers, other tech geeks, to validate, assess, and understand the root cause of vulnerabilities, and work on their mitigation.
- Spread your security knowledge with hundreds of your colleagues!

## Core Requirements

- Proven experience in conducting application security assessments, including planning, testing, and vulnerability reporting;
- Proficiency using security scanners, static code analyzers, and debuggers;
- Understanding of networking and cryptographic fundamentals such as TLS, PFS, PKI, and digital certificates.
- Demonstrated ability to perform manual security code audits;
- Previous experience with fuzzing, reverse engineering, and exploit development.
- Experience working with application developers to validate, assess, understand the root cause, and mitigate vulnerabilities;
- Strong knowledge in application and infrastructure testing methodologies;
- Knowledge in secure architecture design and assessment (manual approach to penetration testing);
- Bonus points for community contributions like public CVEs, bug bounty recognition, open-source tools, blogs, certificates etc.
