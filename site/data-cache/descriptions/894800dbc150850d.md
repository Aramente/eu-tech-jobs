Fortune and Fortune Media IP Limited are not affiliated with, and do not endorse products or services of GitLab.</em></p></div><h3>An overview of this role</h3>
<p>As a <span class="textBold">Backend Engineer</span> on the Secret Detection team, you'll help protect sensitive data by building the services, scanning workflows, and remediation paths that prevent leaked secrets from reaching production. Your work will contribute to the full secret management lifecycle, from push protection to pipeline-based scanning, validation, and auditability, so developers can move quickly without taking on avoidable security risk. This is a strong opportunity if you want to work on security features with clear customer impact, improve detection quality, and help teams act when credentials, API keys, or other secrets are exposed.</p>
<p>You'll focus on backend systems that power Secret Detection across GitLab's DevSecOps platform, working closely with product management and engineering peers in an async-first environment. In your first year, you'll contribute to core product capabilities, improve performance and result quality, and help shape technical direction through code reviews, RFCs, and proof of concepts.</p>
<p><span class="textBold">Some examples of our projects:</span></p>
<ul>
<li value="1">Prevent secret leaks in source code with GitLab Secret Push Protection</li>
<li value="2">Verify validity of secret detection findings</li>
</ul>
<h3>What you’ll do</h3>
<ul>
<li value="1"><span class="textBold">Guide</span> the design and implementation of backend features for GitLab Secret Detection in <span class="textBold">Ruby on Rails</span>, <span class="textBold">GraphQL</span>, and <span class="textBold">Go</span>, delivering capabilities that improve coverage, reliability, or response time for secret detection workflows.</li>
<li value="2"><span class="textBold">Build</span> clean, well-tested, maintainable code that meets GitLab standards for reliability and performance, helping reduce regressions and maintain backend systems at scale.</li>
<li value="3"><span class="textBold">Partner</span> with product management and engineering peers to deliver backend capabilities that improve detection, validation, remediation, and audit trail coverage across the secret management lifecycle.</li>
<li value="4"><span class="textBold">Improve</span> detection quality by reducing false positives, strengthening secret validation workflows, and enabling faster, more effective remediation paths.</li>
<li value="5"><span class="textBold">Contribute</span> to code reviews, RFCs, and proof-of-concept work that guide technical approaches across the Secret Detection category.</li>
<li value="6"><span class="textBold">Identify</span> technical debt and operational inefficiencies, then propose and implement practical improvements.</li>
<li value="7"><span class="textBold">Diagnose</span> performance and optimization issues in backend systems and implement improvements that increase efficiency, scalability, and service reliability.</li>
<li value="8"><span class="textBold">Work</span> effectively in a globally distributed, async-first team while participating in planning, engineering discussions, and pairing when needed.</li>
</ul>
<h3>What you’ll bring</h3>
<ul>
<li value="1"><span class="textBold">Experience</span> building backend applications and services using <span class="textBold">Ruby on Rails</span>, with working knowledge of <span class="textBold">GraphQL</span> and interest in backend-focused product development.</li>
<li value="2"><span class="textBold">Experience</span> designing and delivering secure, maintainable systems that power production web applications at scale.</li>
<li value="3"><span class="textBold">Knowledge</span> of security concepts, common vulnerabilities, mitigation techniques, and secure coding practices.</li>
<li value="4"><span class="textBold">Background</span> developing or working with security tools or products, especially in areas related to code scanning or secret detection.</li>
<li value="5"><span class="textBold">Experience</span> investigating performance issues and improving backend reliability, efficiency, and maintainability.</li>
<li value="6"><span class="textBold">Ability</span> to work closely with cross-functional partners, including product, design, and technical writing, to deliver useful product outcomes.</li>
<li value="7"><span class="textBold">Communicate</span> clearly in writing and in conversation, especially in remote, async-first environments with distributed teams.</li>
<li value="8"><span class="textBold">Bring</span> transferable experience and a willingness to grow into parts of the security or Go stack.</li>
</ul>
<h3>About the team</h3>
<p>The Secret Detection team owns GitLab's <span class="textBold">Secret Detection</span> category, and we build the backend systems and related user workflows that help developers identify and mitigate exposed secrets as code is contributed. We work with the broader security product suite while maintaining focused investment in secret scanning quality, validation, remediation, and developer experience. Our work spans <span class="textBold">Rails</span> and <span class="textBold">Go</span> services, and we work primarily asynchronously across time zones as a globally distributed team. Current opportunities include expanding coverage across the secret management lifecycle and improving result quality across the findings our tools detect.
