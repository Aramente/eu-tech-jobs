<p>The Infrastructure Security team makes it simple and easy for engineers to make secure choices while developing world-class software at high speed. Security is built in by default—seamless and effortless—making engineers’ lives better and Klaviyo more secure.</p>
<p>Our work sits at the intersection of security, infrastructure, and software engineering. We build cloud-native platforms and services that are reliable, secure, and performant at global scale, forming the foundation that Klaviyo engineers rely on every day.</p>
<p>This role is for engineers who enjoy building modern distributed systems, care about security and reliability, and want to have real impact across a large engineering organization.</p>
<h3><strong>How you’ll make an impact</strong></h3>
<p>As a Software Engineer on the Infrastructure Security team, you will build and operate the systems that help Klaviyo engineers ship secure software with confidence. You will:</p>
<ul>
<li>Design, build, and maintain cloud-native security services used across Klaviyo</li>
<li>Own meaningful components and services end to end, from implementation through production operation</li>
<li>Improve the availability, scalability, latency, and efficiency of infrastructure security systems</li>
<li>Collaborate with senior engineers on system design and architecture, contributing ideas and technical solutions</li>
<li>Identify performance, reliability, and security issues in distributed systems and drive improvements</li>
<li>Work extensively with technologies such as Python, Golang, AWS, Kubernetes, Terraform, and modern data stores</li>
<li>Participate actively in design reviews, code reviews, and whiteboarding sessions</li>
<li>Contribute to operational excellence through monitoring, alerting, and incident response</li>
<li>Participate in an on-call rotation with a focus on learning, prevention, and reducing operational toil</li>
<li>Build tooling, automation, and documentation that improve developer experience and security posture</li>
<li>Partner with product-facing and platform engineers to ship impactful, secure solutions</li>
</ul>
<h3><strong>Who you are</strong></h3>
<ul>
<li>You have solid experience building and operating <strong>cloud-native, distributed systems</strong> in production</li>
<li>You are comfortable writing production-quality code in a language such as <strong>Python or Go</strong></li>
<li>You have hands-on experience with <strong>AWS (or a similar cloud provider)</strong> and understand managed services, networking, and IAM concepts</li>
<li>You have experience working with <strong>containers and orchestration platforms</strong> such as Kubernetes</li>
<li>You are comfortable owning services and features independently, while knowing when to ask for feedback or support</li>
<li>You understand the fundamentals of scalable, multi-tenant architectures and secure system design</li>
<li>You care about reliability, performance, observability, and security—not just shipping features</li>
<li>You are comfortable participating in on-call and responding to production issues</li>
<li>You communicate clearly, collaborate well, and enjoy working through technical problems as a team</li>
<li>You regularly use modern developer tooling, including AI-powered tools, to improve productivity and code quality</li>
</ul>
<h3><strong>Nice to have</strong></h3>
<ul>
<li>Experience building or integrating security-focused systems (e.g. secrets management, IAM, authn/z, policy enforcement)</li>
<li>Familiarity with service meshes, API gateways, or zero-trust architectures</li>
<li>Experience improving performance in distributed systems or debugging complex multi-service workflows</li>
</ul>
<h3><strong>Tech Stack</strong></h3>
<p>Klaviyo’s platform is primarily built with Python and React and runs on AWS. Engineers join us from a wide range of technical backgrounds and are supported in learning our stack. Core technologies include:</p>
<ul>
<li>Python / Django / FastAPI</li>
<li>MySQL / Redis / Memcached</li>
<li>RabbitMQ / Celery / Apache Kafka / Apache Pulsar</li>
<li>AWS / Terraform / Kubernetes</li>
</ul>
