<p><strong>About the Role</strong></p>
<p>As a Fullstack engineer, you should be comfortable with both backend (Preferably Golang) and frontend (React, Next). You will be primarily working on shipping new features like new integrations, enabling more machine learning API’s, and building complex workflows. You will also work on optimizing response times, building features that will scale to 100’s of millions of documents we process every month.</p>
<p><strong>What we expect from you -&nbsp;</strong></p>
<ul>
<li>B.Tech/B.E from a reputed educational institute.</li>
<li>Agility in shipping features</li>
<li>Good at code design and architecture</li>
<li>Very good communication</li>
<li>Backend experience - Preferably golang, python</li>
<li>Databases - Understanding of data modeling for Nosql preferably cassandra</li>
<li>Strong fundamentals in OOP design patterns</li>
<li>Basic understanding of Devops</li>
<li>Curiosity and Willingness to learn new things while solving a challenging problem</li>
<li>Learning and incorporating best practices in software development, security and design/architecture</li>
</ul>
<p><strong>Some of the interesting features we have shipped in backend</strong></p>
<ul>
<li>Compile python code into C which could be imported into golang and then shipped as binary for on premise systems</li>
<li>Autoscale GPU dependent services with kubernetes with a custom metric</li>
<li>Displaying machine learning metrics in simplified ways to end users so they can act based on those metrics</li>
<li>Building large number and variety of integrations with relatively generic interface like salesforce, quickbooks, RPA's, external databases</li>
<li>Process large number of files in highly distributed manner in golang</li>
</ul>
<p><strong>Some of the interesting things we have shipped in frontend</strong></p>
<ul>
<li>Ability for users to annotate documents so AI can learn which fields to extract</li>
<li>Displaying machine learning metrics in simplified ways to end users so they can act based on those metrics</li>
<li>Letting users build complex visual workflows around our API in our product.</li>
