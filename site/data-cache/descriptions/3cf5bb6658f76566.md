<p>At JetBrains, we are building an AI-native platform for software development that connects developer workflows, team-level collaboration, and organizational control in a single coherent system. Designed to work across teams, tools, and environments, this platform serves as the execution and governance layer for AI-driven development, with security, policy, and auditability built in from day one.&nbsp;</p>
<p>We are now looking for a Head of Security to own the security strategy and execution for the agentic platform and its core services (governance, identity, agent execution, integrations, telemetry, and billing surfaces). This is a senior leadership role that combines product security, cloud and platform security, and enterprise trust-building. You will partner closely with product and engineering leaders to ensure we can ship fast while meeting the security expectations of regulated and security-conscious organizations.</p>
<h2><strong>Key responsibilities</strong></h2>
<h3><strong>Platform security engineering and operational readiness</strong></h3>
<ul>
<li>Build and run an actionable security program, taking responsibility for vulnerability management, security testing, incident response playbooks, on-call interfaces, and secure SDLC practices aligned with the platform’s release cadence.</li>
<li>Define and implement security controls for platform foundations, such as identity and organizational model, integrations, telemetry, billing surfaces, and administrative UX.</li>
<li>Lead readiness for enterprise security reviews, ensuring security documentation, security posture transparency, and customer-facing security assurance artifacts.</li>
</ul>
<p>Success criteria:</p>
<ul>
<li>Implementation of a repeatable security review process that scales with releases (not dependent on heroics).</li>
<li>Measurable reduction in critical security risks across the platform’s surface.</li>
</ul>
<h3><strong>Customer trust, compliance enablement, and field partnership</strong></h3>
<ul>
<li>Partner with Sales, Solutions, and Professional Services teams to unblock early design partners and enterprise rollouts with credible security answers and practical mitigations.</li>
<li>Drive a pragmatic compliance path (with enterprise-grade expectations) that includes security control mapping, audit support, and customer security questionnaires, at speed.</li>
<li>Act as a security-facing executive in customer conversations when needed.</li>
</ul>
<p>Success criteria:</p>
<ul>
<li>Security becoming an adoption enabler, with fewer blocked deals or PoCs due to security concerns, as well as faster security sign-offs.</li>
<li>Implementation of a clear enterprise-grade security baseline for the platform that supports scale.&nbsp;</li>
</ul>
<h3><strong>Leadership and team building</strong></h3>
<ul>
<li>Build and mentor a lean security team (platform security and product security), with strong relationships with infrastructure, identity, and agent runtime teams.</li>
<li>Implement decision frameworks that clarify what must be centralized vs delegated, when to block vs mitigate, and how to ship securely in an early-stage product environment.</li>
<li>Establish security KPIs that reflect real risk reduction and adoption readiness (not vanity metrics).</li>
</ul>
<p>Success criteria:</p>
<ul>
<li>A high-trust, high-velocity security organization that product and engineering teams want to partner with.</li>
<li>Security priorities that are clearly tied to platform milestones and customer adoption.</li>
</ul>
<h2><span style="font-family: helvetica, arial, sans-serif;"><strong>Requirements</strong></span></h2>
<ul>
<li>Senior experience leading platform or product security for a cloud platform, developer tooling platform, or infrastructure product.</li>
<li>Strong hands-on background in threat modeling, secure architecture, identity and access control, secret management, isolation and sandboxing, and supply chain security.</li>
<li>Experience building security programs that support enterprise adoption (security reviews, compliance enablement, incident readiness, etc.).</li>
<li>Comfort operating in early-stage, evolving product contexts, with the ability to define the security system while the product is still being built and help the organization ship with confidence.</li>
</ul>
