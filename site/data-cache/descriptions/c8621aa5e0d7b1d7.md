<h4><strong>Android CI/CD Engineer</strong></h4>
<p>Adyen is looking for an Android CI/CS Engineer for the In-Person-Payments Tooling team in Amsterdam. This role focuses on CI/CD Engineering, ensuring smooth and efficient software development and deployment processes, with a key area being Android development. You will be instrumental in creating and maintaining CI/CD systems and developing infrastructure tooling, essentially acting as the "glue" that binds code and infrastructure together.</p>
<p>The position involves working on CI/CD for Android terminal and app development. The primary function is developing tools and applications to manage and interact with the Software Development Lifecycle (SDLC) to provide an elevated SDLC experience for the organization. This work involves close collaboration with developers, as well as teams building our on-premises infrastructure, including Kubernetes clusters, the Hashicorp stack, and our private cloud. You will be pivotal in ensuring our software development and deployment processes are smooth and efficient by creating and maintaining CI/CD systems and developing infrastructure tooling.</p>
<h4><strong>What you’ll do</strong></h4>
<ul>
<li>Design and implement CI/CD pipelines for Android applications using tools like GitLab CI, Gradle, KMP, Develocity and Android Native Plugin.</li>
<li>Automate build processes for multiple build variants, flavors, and configurations</li>
<li>Configure build caching and optimization to reduce build times</li>
<li>Configure code quality checks, static analysis, and security scanning</li>
<li>Configure emulators and device farms for automated testing</li>
<li>Integrate monitoring and alerting for tooling health</li>
<li>Set up automated testing frameworks including unit tests, integration tests, and UI tests</li>
</ul>
<h4><strong>Who you are</strong></h4>
<p>You are an <strong>analytical thinker</strong> and a <strong>team player</strong> who thrives at the intersection of mobile development and infrastructure. You don't just write code; you build the systems that allow other engineers to ship their code safely and quickly.</p>
<ul>
<li><strong>A Technical "Glue" Specialist:</strong> You enjoy being the bridge between Android applications and private cloud infrastructure (Kubernetes, Hashicorp stack).</li>
<li><strong>A Gradle Expert:</strong> You have a proven track record of writing and maintaining complex Gradle builds for mobile applications and likely enjoy developing custom plugins to solve build bottlenecks.</li>
<li><strong>An Automation Enthusiast:</strong> You are passionate about "automating everything" to improve Developer Experience.</li>
<li><strong>Autonomous yet Collaborative:</strong> You are comfortable working independently on complex tasks but stay in sync with a team of five dedicated engineers.</li>
<li><strong>SDLC-Focused:</strong> You have a fundamental understanding of the Software Development Lifecycle and can design CI/CD pipelines that handle the scale of a monorepository.</li>
</ul>
<h4><strong>Good to have</strong></h4>
<ul>
<li><strong>AOSP Development:</strong> Experience with Android Open Source Project development.</li>
<li><strong>Advanced Linux &amp; Systems Admin:</strong> Proficiency in Linux troubleshooting (understanding inodes, process management, and resource allocation).</li>
<li><strong>Infrastructure as Code (IaC):</strong> Familiarity with tools like <strong>Terraform</strong> or <strong>Puppet</strong>.</li>
<li><strong>Alternative CI Tooling:</strong> While Gitlab CI is preferred, experience with <strong>Jenkins</strong>, <strong>GitHub Actions</strong>, or <strong>Bitrise</strong> is highly valued.</li>
<li><strong>Advanced Testing &amp; Monitoring:</strong> Experience configuring device farms/emulators for automated testing and integrating monitoring/alerting for tooling health.</li>
<li><strong>Specific Android Tools:</strong> Familiarity with <strong>KMP</strong> (Kotlin Multiplatform), <strong>Develocity</strong>, and specialized testing tools like <strong>Screenshot testing</strong> or <strong>Detekt</strong>.</li>
</ul>
