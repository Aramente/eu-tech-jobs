<p><strong>What You’ll Do</strong></p>
<p>You'll be a key developer on a mission-critical platform, ensuring reliable communication during emergencies.</p>
<ul>
<li>Own end-to-end real-time communication software development for our life-saving platform.</li>
<li>Design and implement a high-performance Media Streaming architecture that achieves low latency and resilience for audio and video feeds.</li>
<li>Develop core RTC components from mobile and specialized clients all the way to the backend infrastructure.</li>
<li>Build and maintain highly scalable communication servers, implementing clustering technologies to handle massive concurrent traffic volumes.</li>
<li>Lead the development of systems for audio processing and handling, as well as robust, compliant cloud-based recording and archiving of all media streams.</li>
<li>Work across diverse environments: mobile platforms, clustered VoIP and Streaming servers, command and control systems, and emergency end-devices.</li>
<li>Actively contribute to projects utilizing state-of-the-art RTC technology, including WebRTC internals, HTTP Streaming, and media frameworks like GStreamer and FFmpeg.</li>
</ul>
<p><strong>What You Bring</strong></p>
<ul>
<li>At least 5 years of experience in backend development using low-level or high-performance languages (e.g., <strong>C++, Rust, Go, or Java</strong>). You should be comfortable thinking about memory management, concurrency, and performance bottlenecks.</li>
<li>You have a proven track record in <strong>either</strong> Real-Time Communication (WebRTC, SIP, Streaming) <strong>or</strong> building complex, high-scale Cloud Infrastructure (AWS/distributed systems).</li>
<li>Demonstrated experience in designing and deploying highly-available, scalable servers and clustering technologies. You understand how to build systems that don't just work, but scale.</li>
<li>A "brave" approach to the tech stack- you aren't afraid to dive into the internals of a library, debug a network protocol, or learn a niche media framework from scratch.</li>
<li>You enjoy solving "hard" engineering problems (think: race conditions, high-latency, or resource-constrained environments).</li>
<li>Exceptional interpersonal abilities and fluent English communication skills.</li>
</ul>
<p>&nbsp;</p>
<p><strong>✨ Bonus Points&nbsp;</strong></p>
<ul>
<li>Familiarity with Media Frameworks (GStreamer, FFmpeg) or protocols (RTP, RTSP, SIP, WebRTC).</li>
<li>Experience with Docker, Kubernetes, and designing microservices-based solutions.</li>
<li>A solid understanding of networking fundamentals (TCP/UDP, TLS, packet loss mitigation).</li>
<li>Experience with profiling tools and monitoring high-throughput systems.</li>
</ul>
